## Plan: V2 Assertion Generalization

Standardize the POC around one generic assertion primitive and a reusable harness-state shape, while keeping the Godot evidence loop, artifact contract, and game/test boundary simple.

The goal of v2 is not a new runner architecture. The goal is to stop adding one bespoke verifier operation per interaction and instead introduce one reusable value assertion that can target data already captured in checkpoints.

**Goals**
1. Keep `assert_value` as the one assertion primitive used by the POC scenarios.
2. Resolve values from named checkpoints using a stable dot-path syntax.
3. Support a small initial comparator set that covers most near-term checks.
4. Prefer enriching harness-observed state over adding verifier-specific assertion wrappers.
5. Standardize reusable semantic node facts so UI and scene scenarios can assert on `harness_state.nodes.<alias>.*` instead of ad hoc fields.

**Non-Goals**
1. Do not change the game/test boundary.
2. Do not replace the local runner, artifact contract, or evidence layout.
3. Do not add Playwright, web export, CI, screenshot diffing, or generalized agent orchestration.
4. Do not add new scenario-specific verifier assertion wrappers unless the generic model proves insufficient.
5. Do not allow the local artifact tree to grow without bounds during normal iteration.

**V2 Contract**
1. `assert_value` reads from one named checkpoint using `checkpoint` and `path`.
2. `path` is dot-separated and resolves relative to the checkpoint root object stored in `summary.json`.
3. Numeric array indexing is allowed through numeric path segments such as `tracked_nodes.1.path`.
4. Initial comparators are: `eq`, `neq`, `gt`, `gte`, `lt`, `lte`, `contains`, `starts_with`, and `ends_with`.
5. Unsupported comparators or incompatible comparator/value type pairs are treated as scenario runtime errors.
6. Missing checkpoints or missing resolved values are treated as assertion failures with explicit evidence in the summary and event log.
7. Harnesses may expose reusable semantic node maps under `harness_state.nodes` and derived measurements under `harness_state.metrics`.

**POC Rules**
1. `move_up_smoke.json` and `pause_menu_focus.json` are the canonical scenarios.
2. Both scenarios should use `assert_value` against checkpoint facts rather than legacy verifier-specific assertion ops.
3. Harnesses should expose semantic maps under `harness_state.nodes` and derived measurements under `harness_state.metrics` when useful.
4. Scenario files may still keep top-level convenience fields in `harness_state`, but new assertions should prefer the reusable semantic maps.

**Implementation Steps**
1. Lock the v2 schema in repo docs. Depends on nothing.
   Add `plans/v2.md` as the reference plan and update the scenario format spec to define `assert_value`, path resolution, and comparator rules.
2. Extend the verifier with generic value resolution. Depends on step 1.
   Add a checkpoint value resolver and comparator engine under `test/scripts/verifiers/scenario_verifier.gd`.
3. Extend the scenario driver. Depends on step 2.
   Add `assert_value` handling in `test/scripts/drivers/scenario_driver.gd` and keep assertion results flowing into `summary.json` the same way as v1 assertions.
4. Enrich the movement harness and migrate the movement scenario. Depends on steps 2 and 3.
   Expose a derived upward displacement value through `harness_state` and migrate `move_up_smoke.json` to `assert_value` instead of `assert_position_delta`.
5. Add a shared harness-state helper. Depends on steps 2 and 3.
   Centralize reusable node-fact extraction so harnesses can publish consistent semantic maps under `harness_state.nodes`.
6. Migrate the pause-menu scenario to the same generic model. Depends on steps 2, 3, and 5.
   Reuse the pause-menu harness but verify visibility, focus, enabled state, and text through `assert_value` against semantic node facts.
7. Remove the legacy assertion surface. Depends on steps 4 through 6.
   Delete redundant scenario-specific assertion ops from the driver and verifier once the canonical scenarios no longer depend on them.
8. Add bounded artifact retention. Depends on steps 1 and 3.
   Prune obsolete scenario artifact folders and keep only a configurable number of recent runs per active scenario.
9. Prove the cleaned-up generic model. Depends on steps 7 and 8.
   Run the canonical movement and pause-menu scenarios through the local runner and confirm both pass with full artifact bundles.

**Relevant Files**
- `/plans/v2.md` — canonical v2 plan for assertion generalization.

- `/plans/specs/artifact_schema.md` — artifact summary expectations for generic assertion results.
- `/test/scripts/harness_support/harness_state_helpers.gd` — shared helper for reusable semantic node facts.
- `/test/scripts/harness_controllers/movement_harness_controller.gd` — derived movement facts exposed through `harness_state` for generic assertions.
- `/test/scripts/harness_controllers/pause_menu_harness_controller.gd` — semantic node facts exposed for reusable UI assertions.
- `/test/scripts/verifiers/scenario_verifier.gd` — generic checkpoint value resolution and comparator engine.
- `/test/scripts/drivers/scenario_driver.gd` — generic scenario execution and `assert_value` handling.
- `/test/scenarios/move_up_smoke.json` — migrated movement scenario using `assert_value`.
- `/test/scenarios/pause_menu_focus.json` — canonical pause-menu scenario using semantic node facts and `assert_value`.
- `/tools/run_scenario.ps1` — local runner with bounded artifact retention.
- `/tools/prune_artifacts.ps1` — manual and runner-invoked artifact pruning tool.

**Verification**
1. Run the migrated `move_up_smoke.json` and confirm pass.
2. Run the canonical `pause_menu_focus.json` and confirm pass.
3. Inspect the migrated movement summary and confirm the `verification` object records `harness_state.actor_upward_displacement` with the expected comparator and value.
4. Inspect the pause-menu summary and confirm the `verification` object records semantic node paths such as `harness_state.nodes.pause_menu.visible` and `harness_state.nodes.resume_button.focused`.
5. Confirm obsolete artifact folders are pruned and only the latest configured runs remain per active scenario.

**Decisions**
- Keep v1 as the proven baseline and treat v2 as the cleaned-up generic POC layer.
- Prefer one generic assertion primitive over a growing list of bespoke assertion ops.
- Keep the first v2 scope intentionally small: checkpoint path lookup, semantic harness state, and a limited comparator set only.
