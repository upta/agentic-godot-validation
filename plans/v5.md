## Plan: V5 Suite Hardening

Build on the v4 scenario breadth by turning batch execution into a retained evidence source and by adding repeat mode for local soak or flake detection. The goal is to harden the suite contract without widening the verifier API or introducing CI yet.

**Goals**
1. Emit a retained suite artifact for every batch run under `artifacts/suites/<timestamp>/suite.json`.
2. Surface a stable latest-suite manifest so the most recent batch run is easy to inspect without scanning the filesystem.
3. Add repeat mode to `tools/run_all_scenarios.ps1` so the full suite can be run multiple times in one invocation.
4. Record flaky scenarios at the suite level when the same scenario both passes and fails across repeated iterations.
5. Keep suite retention bounded the same way per-scenario retention is already bounded.

**Non-Goals**
1. Do not add CI or cloud orchestration yet.
2. Do not expand the scenario driver or verifier surface for suite execution.
3. Do not replace per-scenario artifacts with suite-only artifacts.
4. Do not add screenshot diffing or external reporting infrastructure.

**V5 Contract**
1. `tools/run_all_scenarios.ps1` should write a retained suite summary for every invocation.
2. `tools/run_all_scenarios.ps1 -RepeatCount <n>` should run the full scenario set `n` times and return non-zero if any scenario run fails.
3. The suite summary should record per-iteration results and per-scenario aggregates across all repeated runs.
4. A scenario is considered flaky when it both passes and fails within the same repeated suite invocation.
5. `tools/prune_artifacts.ps1` should preserve the `artifacts/suites/` subtree, keep only the latest configured suite runs, and rebuild the suite manifests.
6. `artifacts/latest_suite.json` should always point at the latest retained suite run when one exists.

**Implementation Steps**
1. Lock the v5 plan in repo docs.
   Add this plan and extend the artifact schema for suite-level evidence.
2. Extend the batch runner.
   Add repeat mode, suite artifact writing, and flaky-scenario aggregation to `tools/run_all_scenarios.ps1`.
3. Extend pruning and manifests.
   Teach `tools/prune_artifacts.ps1` to retain suite runs and rebuild `artifacts/suites/index.json` plus `artifacts/latest_suite.json`.
4. Validate the hardened suite loop.
   Run the full suite with `RepeatCount > 1`, inspect the retained suite manifests, and confirm the output stays stable.

**Relevant Files**
- `/plans/v5.md` — canonical v5 plan.
- `/plans/specs/artifact_schema.md` — suite artifact and repeat-mode schema.
- `/tools/run_all_scenarios.ps1` — batch runner with repeat mode and retained suite evidence.
- `/tools/prune_artifacts.ps1` — per-scenario and per-suite retention plus manifest rebuilding.
- `/artifacts/index.json` — retained per-scenario latest runs.
- `/artifacts/suites/index.json` — retained suite history.
- `/artifacts/latest_suite.json` — shortcut to the latest retained suite run.

**Verification**
1. Run `tools/run_all_scenarios.ps1 -RepeatCount 2` and confirm the full suite completes successfully.
2. Inspect `artifacts/suites/<timestamp>/suite.json` and confirm it records per-iteration results and aggregate scenario outcomes.
3. Inspect `artifacts/latest_suite.json` and confirm it points at the latest retained suite artifact.
4. Inspect `artifacts/suites/index.json` and confirm it reflects the configured suite retention.

**Decisions**
- Treat suite execution as a first-class artifact producer rather than terminal-only output.
- Keep repeat mode in the batch runner rather than adding a second soak script.
- Use suite-level flakiness reporting before adding any CI-facing reliability surface.
